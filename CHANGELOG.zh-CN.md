# 更新日志

面向用户的 Rayterm 发布说明。逐 commit 的完整历史在源码仓库里；本文件只
记录每次更新值得留意的内容。

每个版本的二进制和签名见
[GitHub Releases](https://github.com/highfyj/rayterm-releases/releases)。

English: [CHANGELOG.md](CHANGELOG.md)

## v1.0.18 — 2026-06-12

### 修复

- **拖入文件不再清空整个界面。** 之前把 HTML、图片或音频文件拖进
  Rayterm 窗口，webview 会直接导航到那个文件 —— 所有终端界面瞬间被
  文件内容替换，只能重启找回。现在窗口会拒绝无人认领的拖放（显示
  「禁止」光标）；网页终端里的页面照常接收拖拽，后续的拖入上传等
  功能也不受影响。
- **macOS：中文输入法吞字 / 空格双击。**（仅 macOS；Windows 不受
  影响）这是 xterm.js 在 WKWebView 下的上游缺陷，本版在应用层做了
  完整兜底：
  - 中文输入法下按 Shift 临时切英文、或直接输入半角符号时，第一个
    字符不再被吞；
  - 输入法激活时（无论中/英子模式）按空格不再出现两个空格。

## v1.0.17 — 2026-06-03

### 工具集页面 + 一键安装编程字体

Rayterm 新增 **工具集** 页面 —— 点右上角标题栏新增的扳手图标打开。它是
内置小工具的聚合入口，首个工具是**字体安装器**。

- **一键安装编程字体。** 可选 JetBrains Mono、Fira Code、Cascadia Code、
  Hack、Source Code Pro、IBM Plex Mono。Rayterm 会从字体的 GitHub 仓库
  拉取最新 release，带实时进度条流式下载，解压后帮你装好 `.ttf`/`.otf`
  文件 —— 装完即可在终端字体选择器里直接选用。
- **用户 / 系统两种范围。** 只为当前账户安装（无需提权），或系统级为
  所有用户安装。系统级安装会按各平台原生方式请求授权 —— Windows 走
  UAC、macOS 弹授权框、Linux 用 `pkexec`。
- 每个字体都有**已安装徽标**、下载时的逐字体进度条、以及安装路径提示。
  完整支持中英文。

## v1.0.16 — 2026-06-01

### 亮点

**AI 扫描会匹配你的 shell 了。** 运行 AI 扫描生成终端建议时，Rayterm
现在会告诉 agent 这些命令实际将在哪个 OS / shell 下执行 ——
Windows（PowerShell）、macOS（zsh）、Linux 或 WSL（bash）—— 于是生成的
`shell` 和 `script` 字段会用对应平台的路径、引号和内置命令，不再一律
默认 Linux/bash。Windows 工程不会再拿到一堆跑不了的 bash 命令。

**一键查看所有版本。** 设置 → 关于 → 更新 新增 **在 GitHub 查看所有
版本** 链接，作为应用内更新器无法工作时的手动回退 —— 沙箱安装、或公司
网络封了更新器端点的场景。

### 修复

- **Codex AI agent** —— `codex exec` 不再因目录不是 Git 仓库而拒绝启动
  （`--skip-git-repo-check`），并改为只读运行（`--sandbox read-only`），
  消除部分宿主（尤其 Windows）扫描时遇到的沙箱权限报错。

## v1.0.15 — 2026-05-31

### 保存 / 恢复工作区

现在可以把一组打开的终端标签页（含分屏布局）保存成一个命名的**工作区**，
之后一键还原整组。工作区按工程隔离。

- **保存**：侧边栏 *Terminals* 标题栏新增保存图标，点开后列出当前所有
  打开的标签页。勾选一个、几个或全部，起个名字，就随工程一起存下来，
  分屏布局会完整保留。
- **打开**：已保存的工作区显示在工程终端树下方的列表里。双击即可打开；
  弹窗让你选择**重新打开**（关掉当前标签页并切换到该工作区）或**打开
  新的**（在现有标签页基础上追加），并预览每种选择会影响哪些标签页。
  这个列表与终端树相互独立，不会和「双击打开终端」的手势冲突。
- **启动时恢复**：新增 **设置 → 终端 → 启动时恢复上次工作区** 开关。
  打开后，Rayterm 会在退出时记住所有打开的标签页（含分屏），下次启动
  自动重新打开；分屏里临时新建的 shell 也能还原。

## v1.0.14 — 2026-05-25

### 终端双击行为可配置 + 跳转到已打开

侧边栏的终端列表新增一个常被需求的小开关。之前双击终端配置永远是新开
一个 tab —— 想要新 shell 时很顺手，但当你已经开着该终端只是想切过去
时就有点烦。

- **设置 → 终端 → 双击打开新终端**：默认开（保持旧行为）。关闭后，
  双击改为在该配置已经打开的所有实例之间循环切换；如果一个都没开，
  就回退为新建，所以双击永远有反馈。
- 循环顺序按 tab 顺序 + 树内分屏顺序，连按同一行会在所有窗格间稳定
  跳转。
- 终端行的右键菜单现在固定列出 **新建** 和 **跳转到已打开**，无论
  开关如何，两种操作都能一键调用。

## v1.0.13 — 2026-05-21

### Shell 终端预置命令

Shell 类终端的配置里新增 **预置命令** 字段：写在里面的命令会在终端起来
后逐行自动执行，省掉每次手动 `nvm use 18`、`source ~/.proxyrc`、
`conda activate xxx` 的步骤。

- 编辑器以弹窗形式承载，不挤占设置对话框的篇幅；主界面只显示一行
  「已配置 N 条」摘要 + 编辑按钮。
- 单条命令失败不会卡死终端：和 RUN（exec）终端不同，预置命令是当作
  用户敲键盘那样发送的，某行报错只是输出错误信息，shell 仍然能继续
  正常使用。
- 支持 `#` 开头的注释行，方便分组备注。

### VibeCoding / RUN 环境变量

VibeCoding 与 RUN（exec）终端的配置新增 **环境变量** 编辑器，使用键值
对的形式填写：

- 本地终端：变量直接传给子进程，cmd / PowerShell / bash 都通用。
- WSL / SSH 终端：自动转成 `export KEY='VALUE'; ...; <你的命令>` 注入
  到远端 bash 启动命令前。用户无需关心目标系统的不同语法，rayterm
  会按 OS 自动生成兼容形式。
- 弹窗 UI 同预置命令，主界面只占一行摘要。

### 修复

- **误触 Ctrl+R / F5 清空所有终端 UI**：原先 webview 直接走浏览器
  reload，会把整个 React 树清掉，后台 PTY 进程虽然还在跑但前端再也
  接不上，看起来像「所有终端都消失了」。现在生产构建里拦截了
  `F5`、`Ctrl/Cmd+R`、`Ctrl+Shift+R`、`Ctrl+F5` 这些 reload 快捷键；
  开发模式下保留 HMR fallback 不受影响。
- **PowerShell 终端的预置命令出现在 `>>` 续行提示后不执行**：PTY 里
  Enter 实际上是 CR (`\r`)，PSReadLine 把 `\n` 当作缓冲区内的软换行。
  把行分隔符改成 `\r` 之后 PowerShell 也能正常按行提交，bash / zsh
  的 readline 都接受。

## v1.0.12 — 2026-05-18

### 工程管理器与星标工程

左侧侧边栏的工程列表现在**只显示加了星标的工程**，常用工程不再被一长串
列表淹没。

- 右键点击侧边栏的任一工程，弹出菜单：**重命名**（内联编辑）、**取消
  星标**、**AI 扫描**、**删除**。原本悬停才显示的「AI 扫描 / 删除」
  小图标改放到菜单里，避免误触；「在编辑器中打开 / 编辑设置」两个
  图标仍保留在行内。
- 侧边栏底部新增**全部工程**入口，常驻显示工程总数。点击打开**工程
  管理器**对话框。
- 工程管理器左侧是全部工程列表（支持搜索、星标切换、`+` 新建）；右侧
  是详情卡片，列出根目录、连接目标、Agent、创建时间、终端清单和子目录
  数量。
- 新建工程默认带星标。升级前已有的工程会被视为已加星，不会因为升级而
  从侧边栏消失。

### AI 扫描：Claude 计费提醒

把 AI 扫描 agent 切到 **Claude** 时，会弹出一次性提醒：Anthropic 将于
2026-06-15 把 `claude -p` 迁到独立的 Agent SDK 额度池。可以选择继续用
Claude、一键切到 OpenCode，或永久关闭该提醒。

### 修复

- **Windows: 用 OpenCode / Claude / Codex 做 AI 扫描** 之前会报
  `spawn opencode: program not found`，即便 CLI 确实装好了。原因是 npm
  装的 CLI 在 Windows 下其实是 `.cmd` 包装，Rust 的 `Command::new` 不
  查 `PATHEXT`。现在改用 PATHEXT 感知的查找解析绝对路径，shim 能被
  正确找到。
